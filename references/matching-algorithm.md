# Ask Five Engine · 专家匹配算法

## 目标

根据议题内容、议会模式和资源约束，从 `expert-registry.json` 中选择 3-5 位最合适的专家。

## 输入

1. 用户原始议题文本
2. 用户显式指定的专家（可选）
3. 议会模式（默认：共识）
4. 资源约束（可选）

## 输出

按匹配分数排序的专家候选列表，以及每个专家的入选理由。

---

## Phase A：议题解析

从议题中提取以下信息：

| 字段 | 提取方法 |
|---|---|
| `primary_categories` | 关键词映射到预定义类别：战略、产品、设计、技术、成本、内容、传播、组织、投资、风险、个人成长 |
| `keywords` | 议题中的高频/关键名词和动词 |
| `mode` | 用户指定或根据关键词推断 |
| `constraints` | 用户声明的资源约束 |

---

## Phase B：专家评分

对 registry 中每个专家计算 `score`：

```
score = domain_score * 3 + tag_score * 2 + style_score * 1 + bonus - penalty
```

**默认权重**：domain=3，tag=2，style=1。
如果用户明确偏好某类视角（如"多听听技术上的"），可临时调高对应维度权重 ±1。

### 1. Domain 分（权重 3）

```
domain_score = |expert.core_domain ∩ primary_categories| / max(|primary_categories|, 1)
```

- 专家核心领域与议题主类别重叠越多，分数越高
- 最高为 1.0

### 2. Tag 分（权重 2）

```
tag_score = |expert.topic_tags ∩ keywords| / max(|keywords|, 1)
```

- 专家话题标签与议题关键词重叠越多，分数越高
- 最高为 1.0

### 3. Style 分（权重 1）

根据议会模式给风格加分：

| 议会模式 | 偏好风格 |
|---|---|
| 共识模式 | 全局观、长期主义、系统思维 |
| 对抗模式 | 激进 vs 保守、乐观 vs 悲观 |
| 红队模式 | 批判、逆向、风险、 skeptical |
| 陪审团模式 | 独立、理性、数据驱动 |
| 预审模式 | 风险、逆向、失败导向 |

```
style_score = |expert.style ∩ preferred_styles_for_mode| / max(|preferred_styles_for_mode|, 1)
```

### 4. Bonus

- **用户点名 bonus**：用户明确点名的专家，score 直接设为 +∞
- **视角稀缺 bonus**：如果某领域在已选专家中缺失，该领域专家 +0.3

### 5. Penalty

- **同质化 penalty**：如果已选专家与当前专家在 `core_domain` 上重叠超过 50%，-0.2
  - 重叠比例计算：`|交集| / min(|A|, |B|)`
  - 阈值：> 0.5 触发
- **敏感角色 penalty**：敏感角色默认不选，除非用户明确点名或议题属于政治/舆论分析类
  - 敏感角色由 registry 中 `sensitivity: "sensitive"` 标记

---

## Phase C：选择策略

### 默认策略（共识模式）

1. 按 score 排序
2. 选择分数最高的专家
3. 继续选择下一位，但检查同质化 penalty：
   - 如果新专家与已选专家的 `core_domain` 重叠超过 50%，跳过
4. 直到选满 5 人或无可用专家

### 数量规则

| 议题复杂度 | 主类别数 | 专家数量 |
|---|---|---|
| 简单 | 1 个 | 3 人 |
| 中等 | 2 个 | 4 人 |
| 复杂 | ≥3 个 | 5 人 |

复杂度判断：
- 涉及 ≥3 个主类别 → 复杂
- 涉及 2 个主类别 → 中等
- 涉及 1 个主类别 → 简单

### 排序与选择流程

1. 按 `score` 降序排列所有候选专家
2. 取 score 最高者加入 panel
3. 依次考虑下一位候选：
   - 如果与 panel 中任一专家的 `core_domain` 重叠 > 50%，应用 -0.2 penalty 后重新排序
   - 如果 penalty 后仍排第一，则加入
4. 重复直到达到复杂度对应的 panel 人数上限，或没有更多候选
5. 如果候选耗尽，返回当前 panel 并提示数量不足

### 特殊情况

- **可用专家 < 3**：继续召开，但明确提示"视角不足，建议补充 expert skill"
- **可用专家 = 0**：进入空 registry onboarding 流程
- **用户点名专家不足 3 人**：在点名专家基础上再按分数补充

---

## Phase D：输出入选理由

对每个入选专家，输出：

```
- 专家名
- 匹配分数：0.0 - 1.0（点名专家显示 "用户指定"）
- 入选理由：覆盖了 [领域/标签/风格] 视角
- 风险：可能与 [另一位专家] 在 [某点] 上存在张力
```

---

## 示例

**议题**：我们要不要做一条 TikTok 短视频矩阵？

**解析**：
- primary_categories: ["内容", "传播", "成本"]
- keywords: ["TikTok", "短视频", "矩阵", "做不做"]
- mode: 共识

**候选专家评分（示意）**：

| 专家 | domain_score | tag_score | style_score | 总分 | 是否入选 |
|---|---|---|---|---|---|
| MrBeast | 0.67 | 1.00 | 0.33 | 3.67 | ✅ |
| 张一鸣 | 0.33 | 0.50 | 0.33 | 2.00 | ✅ |
| 乔布斯 | 0.33 | 0.25 | 0.00 | 1.25 | ❌（同质化，产品视角已有） |
| 马斯克 | 0.33 | 0.00 | 0.00 | 1.00 | ❌ |
| 芒格 | 0.33 | 0.00 | 0.33 | 1.33 | ✅ |

最终选出：MrBeast、张一鸣、芒格。
