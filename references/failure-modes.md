# Ask Five Engine · 失败模式与 Fallback

## 通用失败模式

| # | 触发条件 | 一线修复 | 仍失败兜底 |
|---|---|---|---|
| 1 | 用户问题模糊 | 回问 1 个澄清问题：「你最终要做什么决定？」 | 按「战略/产品」默认分类继续 |
| 2 | 推荐专家 skill 未安装 | 使用 `references/expert-fallback-profiles.md` 通用 fallback | 从 panel 中移除该专家，换同类别备选 |
| 3 | 用户拒绝推荐专家名单 | 问用户想换谁 / 想加谁 | 按默认 panel 继续，并说明理由 |
| 4 | 输出超过长度限制 | 每位专家发言压缩到 3 句话 | 只保留秘书处建议和决策卡 |
| 5 | 产出物生成失败 | 降级为纯会议纪要 | 只输出行动清单 |
| 6 | 专家视角同质化冲突 | 强制移除重复类型，加入对立视角 | 提示用户「这些人视角太像，是否需要对抗模式？」 |
| 7 | 子 agent / 评委不可用 | 主 agent 自行评分并标注 `dry_run` | 跳过效果维度，仅做结构评分 |
| 8 | 用户指定敏感场景 | 主动拒绝并说明边界 | 结束会议 |

## Engine 专属失败模式

| # | 触发条件 | 一线修复 | 仍失败兜底 |
|---|---|---|---|
| 9 | `expert-registry.json` 不存在或 JSON 解析失败 | 提示用户检查文件格式；输出最小 registry 模板 | 进入空 registry onboarding 流程 |
| 10 | Registry 中 `experts` 数组为空 | 输出 onboarding 信息，引导安装 expert pack | 等待用户安装后再继续 |
| 11 | Registry 条目缺少必需字段（id/path/core_domain） | 跳过该条目并提示「某条目格式不兼容」 | 忽略该条目，继续处理其他专家 |
| 12 | Expert skill 的 SKILL.md 不存在或无法读取 | 标记为 simulated，使用通用 fallback 画像 | 从 panel 中移除该专家 |
| 13 | Expert skill frontmatter 不兼容 | 仍尝试读取正文，缺失字段用空数组补全 | 跳过该 expert |
| 14 | 议会模式识别失败 | 默认使用「共识模式」并提示用户 | 按共识模式继续 |
| 15 | 可用专家数量 < 3 | 继续召开，但明确提示「视角不足，建议补充 expert skill」 | 只输出单专家/少专家咨询报告 |
| 16 | 可用专家数量 = 0 | 进入 onboarding 流程 | 结束当前会话 |
| 17 | 用户点名专家不存在于 registry | 告知用户未找到该专家，并推荐同类别备选 | 忽略点名，按算法自动匹配 |
| 18 | `<runtime-skills-dir>` 占位符无法解析 | 尝试默认路径 `~/.hanako/skills/` 和 `~/.codex/skills/` | 提示用户手动设置 `ASK_FIVE_SKILLS_DIR` 环境变量 |
| 19 | 输出模式参数非法 | 默认使用「结论优先模式」并提示 | 按结论优先模式继续 |
| 20 | 秘书处综合建议无法生成 | 只输出专家发言摘要 | 输出「当前信息不足以给出建议，请补充...」
