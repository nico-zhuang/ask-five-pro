## P1: 我们的产品有 20 个功能需求，团队资源有限，应该优先做哪个？

我要弄清楚的是：在有限资源下，哪些功能对当前目标用户和关键业务指标的影响最大。

调查清单：
- [ ] 将 20 个需求按用户价值、实现成本、技术依赖、风险分类
- [ ] 访谈核心用户或分析行为数据，确认真实痛点排序
- [ ] 了解团队人效、技术债、交付周期等硬约束
- [ ] 对齐公司本季度北极星指标

调查结论：
- 现状是：需求数量多但缺乏与业务目标和用户事实的对应。
- 关键约束是：资源有限，必须让每一人日都指向可验证的业务结果。
- 我之前不知道但现在知道的是：仅凭列表无法排序。
- 基于以上，我的判断是：先执行低成本、高用户覆盖、能验证商业模式的 1-2 个功能，其余放入待验证池，用数据决定是否继续。

## P2: 这家公司值得投资吗？

我要弄清楚的是：该公司的真实经营状况、护城河和估值是否匹配。

调查清单：
- [ ] 获取近 3 年财务报表、现金流和合同履约情况
- [ ] 一线使用其核心产品/服务，观察用户体验与留存
- [ ] 调研行业竞争格局、政策风险、替代品威胁
- [ ] 访谈客户、离职员工、供应商，获取非公开视角

调查结论：
- 现状是：投资决策需要的事实链尚未补齐。
- 关键约束是：如果无法获取第一手资料，我就没有发言权。
- 我之前不知道但现在知道的是：公开叙事往往与一线事实存在温差。
- 基于以上，我的判断是：若营收质量差、经营性现金流持续为负且没有清晰壁垒，当前估值大概率不划算；但在调查完成前，我不下最终结论。

## P3: 我的视频播放量上不去，标题是『AI编程的10个技巧』，怎么改？

我要弄清楚的是：目标观众在什么场景下会点击，以及当前标题为何没有触发这个行为。

调查清单：
- [ ] 打开平台热榜，收集同领域播放量前 20 的标题与评论区
- [ ] 分析爆款标题的关键词结构、情绪钩子与结果承诺
- [ ] 采访 5-10 位目标观众，记录他们看到标题时的第一反应
- [ ] 用 A/B 测试发布 3-5 个候选标题，比较点击率

调查结论：
- 现状是：标题过于通用，缺少具体痛点和点击动机。
- 关键约束是：播放量上不去不一定全是标题问题，但标题是最快验证的变量。
- 我之前不知道但现在知道的是：观众更可能为“解决我当下的具体困境”点击。
- 基于以上，我的判断是：可试改为类似“为什么你学了 AI 编程还是写不出项目？｜10 个真正有用的技巧”这种具体场景 + 情绪共鸣的结构，再用数据验证。

## P4: 我们是一家初创公司，想做一个 AI 硬件产品，该从哪下手？

我要弄清楚的是：团队能力边界、真实需求场景和供应链约束。

调查清单：
- [ ] 盘点团队在硬件、算法、供应链、渠道上的短板
- [ ] 深入 3-5 个潜在用户场景做田野观察，记录真实行为
- [ ] 调研核心元器件成本、供应链交期、合规认证周期
- [ ] 访谈 3 位已量产 AI 硬件的创始人，复盘踩坑经验

调查结论：
- 现状是：产品形态尚未经过事实验证。
- 关键约束是：硬件一旦开模，试错成本远高于软件。
- 我之前不知道但现在知道的是：需求不是问出来的，而是在真实场景中观察出来的。
- 基于以上，我的判断是：不要先定义硬件形态，先用最低成本方案验证“谁、在什么场景、愿为什么问题付费”，再反推硬件规格。

## P5: 我们拿到 3 个护肤品牌的 8000 条小红书笔记和评论，想分析沟通策略差异。

我要弄清楚的是：三个品牌在小红书上的实际话语体系、用户反馈与转化路径差异。

调查清单：
- [ ] 按品牌、笔记类型、评论情感做数据清洗与分层
- [ ] 提取高频卖点词、场景词、情绪词，进行品牌间对比
- [ ] 抽样阅读原始笔记与评论，区分事实描述与营销话术
- [ ] 访谈 10 位目标消费者，验证关键词是否真正引发共鸣

调查结论：
- 现状是：数据量大但尚未完成结构化编码。
- 关键约束是：沟通策略差异必须建立在用户真实反应上，而不是品牌自说自话。
- 我之前不知道但现在知道的是：同样功效词，不同品牌可能绑定完全不同的情绪身份。
- 基于以上，我的判断是：先做“功效-情感-身份认同”三维标注，再看每个品牌在各维度上的占比与差异，然后再下策略判断。

## P6: 我面对一个重要决策，但信息不够，该怎么办？

我要弄清楚的是：信息缺口具体在哪里，以及哪些信息能真正改变决策。

调查清单：
- [ ] 写下决策标准和两种结果各自的代价
- [ ] 列出“必须知道”与“知道更好”两类信息
- [ ] 区分哪些信息可通过调研获取、哪些只能靠推断
- [ ] 设定信息收集截止时间和可接受的止损线

调查结论：
- 现状是：决策风险主要来源于信息缺口，而不是选项本身。
- 关键约束是：时间有限，不可能无限收集信息。
- 我之前不知道但现在知道的是：很多“信息不够”其实是“关键问题没被拆清楚”。
- 基于以上，我的判断是：先暂停拍板，用田野调查补齐能改变判断的关键事实；如果确实无法获取，就选择可逆性最高、事后后悔最小的方案。
