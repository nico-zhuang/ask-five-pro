## P1: 我们的产品有 20 个功能需求，团队资源有限，应该优先做哪个？

我不会先拍脑袋排序。我会把这 20 个需求看成 20 个待认领任务，先启动 3 个并行评估 Agent：一个按「用户痛苦频次」打分，一个按「实现成本/风险」打分，一个按「战略契合度」打分。每个 Agent 输出 1-5 分并附一条证据，最后汇总成加权分：影响力 × 置信度 ÷ 成本。前三名进入 Sprint，其余进 backlog。如果缺少用户行为数据或成本估算，我会先标红这几个需求，不强行推进。

## P2: 这家公司值得投资吗？

我会把它拆成 5 个独立侦察任务并行跑：护城河、财务健康度、团队执行力、市场空间、尾部风险。每个 Agent 只盯一个维度，输出「事实 + 评分 + 置信度」。然后再派一个 synthesis Agent 找矛盾：比如财务好但护城河弱，市场大但团队没跑过。如果关键数据拿不到（收入、留存、创始人背景），我不会给 Yes/No，只会说「在当前信息密度下，这项投资的置信度是 X，需要补齐以下三项证据」。

## P3: 我的视频播放量上不去，标题是『AI编程的10个技巧』，怎么改？

原标题太像知识罗列，没有触发点击的钩子。我会让 5 个标题 Agent 并行生成不同方向：数字反差型、身份标签型、痛点场景型、反常识型、结果承诺型。比如可以改成「工作 3 年后才发现，AI 编程真正提速的是这 3 个习惯，不是 10 个」，或者「月薪 3 万和 1 万程序员，用 AI 编程差在哪」。先批量生产 20 个标题，挑 3-5 个做 A/B 测试，看 2 小时点击率，再决定主标题。

## P4: 我们是一家初创公司，想做一个 AI 硬件产品，该从哪下手？

别从硬件开始。先做场景：我会派 4 个 Agent 并行调研，一个找用户每天重复 3 次以上的具体动作，一个拆 BOM 和供应链可行性，一个看竞品为什么死，一个设计最小可演示单元。目标不是造一个「AI 硬件」，而是找到一个「不用 AI 就痛、用了 AI 就爽」的 5 分钟场景。选定场景后，先用软件模拟 80% 体验，验证需求再立项开模。信息不足时，我会要求先完成 30 个用户访谈。

## P5: 我们拿到 3 个护肤品牌的 8000 条小红书笔记和评论，想分析沟通策略差异。

数据够大，直接上蜂群。我会把 8000 条按品牌分三组，每组配 3 个 Agent：一个提取「人设/价值观」，一个提取「高频痛点词与解决方案话术」，一个分析「评论区情绪与购买意向词」。然后再派一个横向对比 Agent 输出矩阵：品牌 A 主打成分党，品牌 B 主打情绪疗愈，品牌 C 主打平价功效。最后给出一个空白机会，比如「高端敏感肌场景」是否被遗漏。注意要先统一分词和清洗规则，否则对比会失真。

## P6: 我面对一个重要决策，但信息不够，该怎么办？

信息不够就别急着投票。我会先把这个决策写成一个问题树：根节点是决策，叶子节点是「我还不知道什么」。然后给每个未知项派一个 Agent 去补齐：有的找数据，有的找人聊，有的专门负责挑刺（红队）。规定一个 Deadline，到期后所有 Agent 提交证据和置信度。如果关键未知项无法验证，就设计一个低成本实验先跑，而不是 all in。我的原则是：决策质量 = 信息密度 × 反对意见强度，不是谁先发言谁赢。
