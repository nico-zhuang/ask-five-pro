# Brain Crew LLM 批量实测报告

## 测试规模

- **专家数量**：30/30
- **Prompt 数量**：6 个
- **总输出数**：180 条
- **生成方式**：subagent 读取专家 SKILL.md 后按角色规则生成

## Prompt 列表

1. 产品优先级：20 个功能需求如何取舍
2. 投资决策：这家公司值得投资吗
3. 内容 CTR：视频标题怎么改
4. AI 硬件：初创公司该从哪下手
5. 品牌策略：3 个护肤品牌的小红书数据分析
6. 决策困境：信息不够时怎么办

## 评估规则

每条输出按 5 项轻量规则评分：

- 是否破角色（-4）
- 是否输出工程标注（-2）
- 是否过度 hedging（-2）
- 是否使用第一人称（-2）
- 长度是否合适（-1）

## 评估结果

- **平均质量分**：7.70/10
- **最高分**：Paul Graham 8.67/10、Warren Buffett 8.67/10
- **最低分**：Darwin 7.0/10、Freud 7.0/10、GWTB 7.17/10

## Darwin 评分更新

`darwin-quick-score.py` 的 `score_measured()` 已更新：

- 读取 `tests/llm-output-evaluation.json`
- 按每位专家的真实输出质量单独给 dim8 分数
- 质量分 0-10 映射到 dim8 的 0-3.5 分加成

最终 Darwin 评分：

- **全量平均**：92.4/100
- **最低分**：张雪峰 86.6
- **30/30 专家 ≥85**

## 发现的问题

1. **中文名专家匹配问题**：dim8 评分早期用名字模糊匹配，导致中文名专家无法关联到 LLM 输出文件。已改为用 `expert_id` 精确匹配。
2. **质量差异较小**：最高 8.67 与最低 7.0 只差 1.67 分，说明评估规则偏宽容，未来需要更严格的角色 voice 评估。
3. **部分专家跨领域表现弱**：当 prompt 不在专家核心领域时，输出质量下降明显（如 Darwin 回答产品/投资决策题）。

## 下一步建议

1. 增加更多领域-specific prompt，减少跨领域失真。
2. 引入人工抽检，对低分输出做二次标注。
3. 把 LLM 批量测试做成 CI 流程，每次修改 expert 后自动跑。
