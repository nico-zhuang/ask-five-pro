# Brain Crew Pack 迁移建议

## 方法

使用 `scripts/compute-overlap-matrix.py` 计算同 pack 内专家在 `core_domain` 和 `topic_tags` 上的重叠度：

```
重叠度 = 0.6 × core_domain 重叠 + 0.4 × topic_tags 重叠
```

## 结论

**当前 30 位专家全部保留在原 pack，本轮不迁移。**

理由：
- 同 pack 最高重叠度仅 0.28（段永平 vs 巴菲特 / 王阳明）
- 没有 ≥0.50 的高重叠配对
- 各 pack 人数未超过建议上限

## 观察

虽然标签层面重叠不高，但概念层面存在潜在重叠区域：

| 重叠领域 | 涉及专家 | 建议 |
|---|---|---|
| 产品战略 | 乔布斯、张一鸣、Paul Graham | 已在 tags 中区分（设计/系统/创业），无需迁移 |
| 投资决策 | 芒格、巴菲特、段永平 | 段永平偏「常识+中国语境」，巴菲特偏「护城河」，芒格偏「多元思维模型」 |
| 内容创作 | MrBeast、X 导师、Naval | 分别覆盖 YouTube 工程化、Twitter/X 思想传播、财富与杠杆，定位清晰 |

## 建议的后续动作

1. 保留 `packs/extension/` 作为未来扩容出口。
2. 收集真实调用数据后，按调用频率二次评估。
3. 如果未来单 pack 超过上限，优先迁出得分最低、调用最少的专家。
